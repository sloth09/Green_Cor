## Chapter 3: Case 2 상세 분석 - 근거지 운송 시나리오

### 3.1 시나리오 정의

#### 3.1.1 Case 2의 두 가지 버전

Case 2는 **부산항에 저장탱크가 없는** 상황을 모델링합니다. 대신 암모니아를 **원산지(근거지)에서 생산/저장**하고, **필요할 때마다 대형 셔틀로 운송**합니다.

**Case 2-1: 여수 → 부산**
```
[여수 암모니아 생산시설]
    ↓ (저장탱크, 30,000~50,000톤)
[대형 셔틀] (5,000~50,000 m³)
    ↓ (86해리, 약 5.73시간)
[부산항 선박들]
```

**Case 2-2: 울산 → 부산**
```
[울산 암모니아 생산시설]
    ↓ (저장탱크)
[대형 셔틀] (5,000~50,000 m³)
    ↓ (25해리, 약 1.67시간)
[부산항 선박들]
```

#### 3.1.2 근본적 운영 차이

Case 1과 Case 2의 **가장 중요한 차이**:

| 구분 | **Case 1** | **Case 2** |
|-----|----------|----------|
| **저장 위치** | 부산 (도착지) | 근거지 (출발지) |
| **셔틀 역할** | 반복 픽업-델리버리 | 한 번의 대량 운송 |
| **한 번 항해** | 5,000 m³ 이하 (소형) | 최대 50,000 m³ (대형) |
| **삼각형 구조** | 출발-도착-출발 | 출발-도착-출발 (거리 증가) |
| **선박당 서빙** | 1척 (콜당) | 여러 척 (한 항해로) |

#### 3.1.3 핵심 개념: "한 번의 항해로 여러 선박에게 주유"

Case 2의 운영 개념은 완전히 다릅니다:

**예시: 25,000 m³ 셔틀**
```
한 번의 왕복 여행:
1. 여수 항구에서 25,000 m³ 적재 (15시간 소요)
2. 부산으로 항해 (5.73시간, Case 2-1 기준)
3. 부산에서 계류 중인 선박 5척에게 각각 5,000 m³씩 주유
   - 선박 1: 5,000 m³ (호스 연결 30분 + 주유 5시간 + 호스 해제 30분)
   - 선박 2: 5,000 m³
   - 선박 3: 5,000 m³
   - 선박 4: 5,000 m³
   - 선박 5: 5,000 m³
4. 여수로 복귀 (5.73시간)

결과: 25,000 m³로 5척의 선박을 주유
비용: 1회 셔틀 항해로 5척 서빙
```

이것이 **Case 2의 효율성 메커니즘**: 큰 셔틀 × 한 번의 항해 = 여러 선박 동시 서빙

### 3.2 선박당 주유 패턴

#### 3.2.1 Case 2-2 예시 (울산 → 부산)

**정상 운영 시나리오**:

```
Timeline (50,000 m³ 셔틀 기준):

날짜 1 (월요일):
- 09:00: 울산 항구에 도착하여 50,000 m³ 적재 시작
- 23:00: 적재 완료 (14시간)

날짜 2 (화요일):
- 00:00: 출발
- 02:40: 부산 도착 (25해리 ÷ 15노트 = 1.67시간)
- 03:00: 선박 A와의 호스 연결
- 03:30: 선박 A 주유 시작
- 08:30: 선박 A 주유 완료 (5,000 m³)

- 09:00: 선박 B와의 호스 연결
- 09:30: 선박 B 주유 시작
- 14:30: 선박 B 주유 완료

... (선박 C, D, E도 동일하게 반복)

- 그날 밤 또는 다음날 오전: 울산으로 복귀 시작
- 다음날 오전: 울산 도착

총 사이클 타임: 약 26시간
서빙된 선박: 10척 (50,000 m³ ÷ 5,000 m³)
```

#### 3.2.2 주유 순서와 조정

실제로는 모든 선박이 동시에 부산에 있지 않으므로:

1. **사전 예약 시스템**: 선박들이 24시간 전에 주유 예약
2. **스케줄링**: 셔틀 운영자가 주유 순서 최적화
3. **대기 시간**: 연료를 기다리는 선박 최소화

**모델의 단순화 가정**:
- 모든 선박이 필요할 때 즉시 주유 가능
- 셔틀 크기 = 여러 선박의 수요 합
- 펌프 용량이 충분하면 병렬 주유 가능

### 3.3 셔틀 크기와 선박 서빙 관계

#### 3.3.1 한 번의 항해로 서빙 가능한 선박 수

```
Vessels_per_trip = floor(Shuttle_Size / Bunker_Volume_per_Call)
                 = floor(Shuttle_Size / 5,000)
```

| 셔틀 크기 (m³) | 한 항해당 선박 수 | 연간 서빙 잠재력 (척) |
|--------------|-----------------|-----------------|
| 5,000 | 1 | 200 (연 200회 항해 가정) |
| 10,000 | 2 | 400 |
| 15,000 | 3 | 600 |
| 20,000 | 4 | 800 |
| 25,000 | 5 | 1,000 |
| 30,000 | 6 | 1,200 |
| 40,000 | 8 | 1,600 |
| 50,000 | 10 | 2,000 |

**관찰**:
- 셔틀이 클수록 **한 번에 더 많은 선박 서빙 가능**
- 하지만 **가동 시간이 더 필요** (적재, 펌핑, 복귀)
- 최적점은 **시장의 선박 밀집도**에 달려있음

#### 3.3.2 "트립 수"의 의미 (Case 2에서는 1)

Case 2에서 **가장 중요한 수정 사항**:

```
Case 1: Trips_per_Call = ceil(Bunker_Volume / Shuttle_Size)
        예: 5,000 m³ / 1,000 m³ = 5 트립

Case 2: Trips_per_Call = 1 (항상!)
        대신: Vessels_per_Trip = floor(Shuttle_Size / Bunker_Volume)
```

**이유**:
- Case 1: 작은 셔틀이 **같은 선박에게 여러 번** 왕복
- Case 2: 큰 셔틀이 **여러 선박에게 한 번의 항해로** 공급

따라서 Case 2의 **y[t]는 "항해 수"**이며, 각 항해는:
```
Delivery_Volume = Shuttle_Size (가득 찬 용량)
= Vessels_per_Trip × 5,000 m³
```

### 3.4 운영 타이밍 계산

#### 3.4.1 Case 2-2 울산 예시

**50,000 m³ 셔틀 + 1,000 m³/h 펌프**:

```
1. 적재 (여수/울산):
   Time = 2 × (50,000 m³ / 1,500 m³/h)
        = 2 × 33.3h = 66.7시간
   (왕복 고려: 실제로는 편도 33.3시간, 하지만 계산에서는 2배)

2. 항해 (일편도):
   Time = 1.67시간

3. 주유 (부산):
   Time = 2 × (50,000 m³ / 1,000 m³/h)
        = 2 × 50시간 = 100시간
   (로딩 50시간 + 언로딩 50시간)

4. 귀항:
   Time = 1.67시간

총 사이클: 66.7 + 1.67 + 100 + 1.67 = 170시간

연간 최대 항해: 8,000시간 / 170시간 = 47회
```

**해석**: 한 척의 50,000 m³ 셔틀로 **연간 최대 47회 항해 가능**

#### 3.4.2 사이클 타임의 주요 구성요소

```
Full_Cycle = 2×Travel_Time + 2×Setup_Time + Pumping_Time

Case 2-1 (여수) 기준:
- Travel: 5.73시간 (편도)
- Setup: 0.5시간 (호스 연결/해제)
- Pumping: 2 × (Volume / Pump_Rate)

25,000 m³ 셔틀, 1,000 m³/h 펌프:
= 2×5.73 + 2×0.5 + 2×(25,000/1,000)
= 11.46 + 1 + 50
= 62.46시간

연간 최대 항해: 8,000 / 62.46 = 128회
```

### 3.5 비용 구조 비교

#### 3.5.1 셔틀 비용

**Case 2에서는 훨씬 큰 셔틀 필요**:

```
CAPEX (자본비):
Shuttle_CAPEX = 61,500,000 × (Size / 40,000)^0.75

- 25,000 m³: 61,500,000 × (25,000/40,000)^0.75 = 42,600,000 USD
- 50,000 m³: 61,500,000 × (50,000/40,000)^0.75 = 68,500,000 USD

고정 OPEX (CAPEX × 5%):
- 25,000 m³: 2,130,000 USD/년
- 50,000 m³: 3,425,000 USD/년

변동 OPEX (연료비):
MCR 값 (로그 외삽):
- 25,000 m³: 2,981 kW
- 50,000 m³: 3,867 kW

연료 소비 (왕복 기준, Case 2-2):
- 25,000 m³: 2,981 kW × 379 g/kWh × 2×1.67h / 1,000,000 = 3.78 ton
- 50,000 m³: 3,867 kW × 379 g/kWh × 2×1.67h / 1,000,000 = 4.91 ton

연료비 (항해당):
- 25,000 m³: 3.78 × 600 = 2,268 USD
- 50,000 m³: 4.91 × 600 = 2,946 USD
```

**특징**: Case 1의 5,000 m³ 셔틀 ($771 연료비/사이클)과 달리, **Case 2는 훨씬 비싼 연료비**

#### 3.5.2 펌프 비용

**문제**: Case 2에서는 펌프 용량이 **병목**이 될 수 있습니다.

```
50,000 m³를 펌프해야 하는데:
- 1,000 m³/h: 50시간 필요 (비현실적, 선박들이 50시간 대기)
- 2,000 m³/h: 25시간 필요
- 4,000 m³/h: 12.5시간 필요 (병렬 펌핑, 2개 펌프)

대형 펌프 CAPEX:
2,000 m³/h 펌프:
Power = 158.73 × 2,000 = 317,460 kW (!)
CAPEX = 317,460 × 2,000 = 634,920,000 USD (!!)

이는 현실적이지 않으므로, 모델은 보통 여러 중간 크기 펌프를 선택:
- 1,000 m³/h × 4개 펌프 = 4,000 m³/h 총 용량
- CAPEX = 505,600 × 4 = 2,022,400 USD
```

**실제 최적해**: Case 2는 **대형 펌프 사용으로 CAPEX 증가**

#### 3.5.3 저장탱크 비용 (근거지에서)

Case 2에서는 **부산에는 탱크 없음**, 대신 **근거지(여수/울산)에만 탱크**:

```
근거지 탱크:
- 용량: 50,000~100,000톤 (대형 셔틀을 위해)
- CAPEX: 50,000톤 × 1,000 × 1.215 = 60,750,000 USD
- 고정 OPEX: 60,750,000 × 3% = 1,822,500 USD/년
- 냉각비: 50,000,000 kg × 0.0378 kWh/kg × 0.0769 USD/kWh = 146,556 USD/년
```

모델에서는 **Case 2의 탱크가 활성화되지 않음** (부산에만 저장 없다고 가정)

### 3.6 최적화 제약식 (Case 2 특화)

#### 3.6.1 수요 충족 (Case 2)

```
y[t] × Shuttle_Size >= Annual_Demand[t]

예: 2030년, 25,000 m³ 셔틀
y[2030] × 25,000 >= 1,905,000 m³
y[2030] >= 76.2 항해

즉, 한 해에 76회 이상의 왕복 항해 필요
```

**비교**:
- Case 1: y[2030] >= 381 (5,000 m³당 콜 단위)
- Case 2: y[2030] >= 76 (25,000 m³당 항해 단위)

Case 2가 **항해 횟수로는 적지만**, 각 항해의 **거리와 에너지는 훨씬 큼**

#### 3.6.2 작업시간 제약 (Case 2)

```
y[t] × 1 (trips_per_call) × Cycle_Duration <= N[t] × 8,000

예: 25,000 m³ 셔틀, 1,000 m³/h 펌프
y[t] × 1 × 62.46 <= N[t] × 8,000
y[t] <= N[t] × 128.1

즉, 한 척의 셔틀은 연간 최대 128회 항해 가능
```

**2030년 요구사항**: 76회 항해 < 128회 가능
→ **한 척의 셔틀로도 충분**

**2050년**: 790회 항해 필요
→ **6~7척의 셔틀 필요** (790 / 128)

### 3.7 Case 2의 경제성 분석

#### 3.7.1 최적 조합의 특성

Case 2의 최적 해는 일반적으로:
- **중형 셔틀** (5,000~25,000 m³)
- **중형 펌프** (600~1,200 m³/h)

**Case 1과의 차이점**:
| 항목 | Case 1 | Case 2 |
|-----|--------|--------|
| 셔틀 크기 | 500~5,000 m³ | 5,000~50,000 m³ |
| 펌프 크기 | 400~1,600 m³/h | 600~2,000 m³/h |
| 셔틀 수 | 0.4~4척 (연간) | 0.8~8척 (연간) |
| 주요 비용 | 운영(연료, 다중 트립) | 자본(대형 선박) |

#### 3.7.2 Case 2-2 (울산) 최적 예시

**최적 조합: 5,000 m³ 셔틀 + 1,000 m³/h 펌프**

```
CAPEX:
- 셔틀: 18,917,000 USD × 1척 (2030년) = 18,917,000 USD
- 펌프: 505,600 USD (2030년)
- 탱크 (부산): 0 USD (없음)
소계: 19,422,600 USD

OPEX (2030년):
- 셔틀 고정비: 945,850 USD
- 펌프 고정비: 25,280 USD
- 셔틀 연료비: 1.284 ton/항해 × 600 USD × 381 항해 = 292,896 USD
- 펌프 에너지비: 574.8 USD/콜 × 381 콜 = 219,058 USD
소계: 1,483,084 USD/년

20년 NPV:
- CAPEX는 Case 1보다 훨씬 낮음 (탱크 비용 없음)
- OPEX도 비슷하거나 낮음
- 일반적인 결과: $1.5~2.2 billion (Case 1보다 20~30% 저렴)
```

#### 3.7.3 Case 2-1 (여수) vs Case 2-2 (울산) 비교

**두 가지는 거의 동일하지만 거리만 다름**:

```
Case 2-1 (여수, 86해리):
- 사이클 타임: 더 김 (항해 시간 5.73시간)
- 연료비: 더 많음
- 연간 가능 항해: 더 적음

Case 2-2 (울산, 25해리):
- 사이클 타임: 짧음 (항해 시간 1.67시간)
- 연료비: 적음
- 연간 가능 항해: 더 많음

경제성: Case 2-2가 20~30% 더 우수
```

### 3.8 Case 2의 주요 특징 요약

| 특징 | 설명 |
|-----|------|
| **공간 구조** | 근거지에서 대량 운송으로 효율성 추구 |
| **인프라 투자** | 부산 탱크 불필요로 초기 투자 최소화 |
| **셔틀 특성** | 대형 (5,000~50,000 m³)으로 대량 운송 |
| **운영 방식** | 낮은 빈도, 높은 용량의 항해 |
| **수요 대응성** | 사전 예약 필수 (탱크 없어서) |
| **비용 구조** | 낮은 CAPEX 대신 긴 항해 거리 증가 |
| **최적 선택 기준** | 선박 밀집도(여수/울산 근접성) vs 수요 규모 |
