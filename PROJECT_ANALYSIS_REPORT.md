## Executive Summary

### 프로젝트 핵심

본 프로젝트는 2030~2050년 20년 동안 부산 Green Corridor에서의 **친환경 암모니아 연료 공급 인프라를 최적화**하는 혼합정수선형계획(MILP) 모델입니다. 핵심 목표는:

1. **수요 예측**: 50척(2030년)에서 500척(2050년)으로 증가하는 암모니아 연료 선박 함대의 **연간 수요를 정확히 산정**
2. **의사결정**: **셔틀 선박의 최적 크기, 개수, 펌프 용량**을 결정하여
3. **비용 최소화**: **20년간의 순현재가(Net Present Cost, NPC)를 최소화**

2050년의 연간 수요는 2030년 대비 **약 10배 증가** (1.9M m³ → 19M m³)하므로, 초기부터 **확장 가능하고 경제적인 인프라**를 설계하는 것이 매우 중요합니다.

### 세 가지 Case의 구조적 차이

모델의 세 가지 Case는 암모니아 공급의 **지리적 위치와 운영 방식**의 근본적인 차이를 반영합니다:

| 항목 | **Case 1: 부산 저장소** | **Case 2-1: 여수** | **Case 2-2: 울산** |
|-----|----------|----------|----------|
| **시나리오** | 부산항에 저장탱크 | 여수 생산→부산 | 울산 생산→부산 |
| **운송 거리** | 항만 내부 (~1시간) | 86해리 (5.73시간) | 25해리 (1.67시간) |
| **부산 저장시설** | 있음 (35,000톤) | 없음 | 없음 |
| **셔틀 크기** | 500~5,000 m³ (소형) | 5,000~50,000 m³ (대형) | 5,000~50,000 m³ (대형) |
| **운영 개념** | 소형 셔틀 × 다중 왕복 | 대형 셔틀 × 단일 항해 | 대형 셔틀 × 빠른 회전율 |
| **초기 투자** | ~$50.6M | ~$16.1M | ~$16.1M |
| **20년 NPC** | ~$2.5~3.5B | ~$1.6~2.3B | ~$1.5~2.2B |

**근본적인 차이**:
- **Case 1**: 작은 셔틀이 저장탱크에서 **여러 번 왕복**하여 선박들에게 연료 공급 (높은 유연성, 높은 초기 비용)
- **Case 2-1/2-2**: 큰 셔틀이 원산지에서 대량 운송 후 **한 항해로 여러 척 동시 서빙** (낮은 초기 비용, 예약 필수)

### 시간 구조의 중요성

각 Case의 **사이클 타임**(완전한 왕복 시간)은 **셔틀이 얼마나 자주 운항할 수 있는가**를 결정하며, 이는 **필요한 셔틀 개수와 비용**에 직접 영향을 미칩니다:

#### Case 1: 부산항 내 고속 회전
```
한 사이클 구성:
- 육상 탱크에서 적재:           3.33시간 (5,000 m³ ÷ 1,500 m³/h)
- 셔틀 편도 이동:               1시간
- 호스 연결 및 퍼징:            1시간
- 벙커링 (부산 내 선박):        5시간 (5,000 m³ ÷ 1,000 m³/h 펌프)
- 호스 분리 및 퍼징:            1시간
- 셔틀 복귀:                    1시간
─────────────────────────
총 사이클:                      11.33시간

→ 연간 최대 가능: 8,000시간 ÷ 11.33 = **706회 운항**
→ 한 척의 셔틀이 매우 높은 빈도로 운항 가능
```

#### Case 2-2: 울산 장거리 운송
```
한 사이클 구성 (5척 서빙 기준):
- 근거지 준비:                  1시간
- 육상 탱크에서 적재:           16.67시간 (25,000 m³ ÷ 1,500 m³/h)
- 울산→부산 항해:               1.67시간
- 부산 진입:                    1시간
- 5척의 선박 서빙 (각 8시간):   40시간
- 부산→울산 복귀:               1.67시간
- 근거지 도착:                  1시간
─────────────────────────
총 사이클:                      63시간

→ 연간 최대 가능: 8,000시간 ÷ 63 = **127회 항해**
→ 각 항해에서 5척 동시 서빙 = 635척 연간 처리
```

**비교**: Case 2-2는 Case 1보다 사이클이 5.6배 길지만, 한 번에 5배 더 많은 용량을 운송하므로 **초기 셔틀 수는 매우 적음**.

```
참고) 시장가 600 USD/ton 기준으로 보면:
- Case 1: 선박당 추가 비용 30~58%
```

### 의사결정 매트릭스

| 상황 | 최적 선택 | 이유 |
|-----|---------|------|
| **이미 부산에 암모니아 시설 있음** | Case 1 | 인프라 재활용, 높은 유연성 |
| **부산 선박 밀집도 높음** | Case 1 | 탱크의 즉각적 대응 가치 |
| **초기 투자 최소화 중요** | Case 2-2 | 67% 낮은 초기 투자 |
| **근처에서 생산 가능** | Case 2 | 운송 거리 단축의 이점 |
| **선박 도착 시간 불규칙** | Case 1 | 탱크의 버퍼링 기능 필수 |

### 모델의 기술적 특징

#### 의사결정 변수 (Decision Variables)
- **x[t]**: t년도 신규 추가 셔틀 수 (정수)
- **N[t]**: t년도 누적 셔틀 수 (정수)
- **y[t]**: t년도 연간 운항/항해 수 (연속)
- **z[t]**: t년도 신규 탱크 수 (Case 1만, 정수)

#### 최적화 제약식 (Constraints)
1. **수요 충족**: 연간 운항 × 용량 ≥ 수요
2. **작업 시간**: 연간 운항 시간 ≤ 셔틀 수 × 8,000시간
3. **탱크 용량** (Case 1만): 셔틀 총 용량 × 안전계수 ≤ 탱크 총 용량

#### 비용 함수 (Cost Function)
```
최소화: NPC = Σ_t [할인율 × (CAPEX(t) + 고정OPEX(t) + 변동OPEX(t))]

구성:
- CAPEX: 셔틀, 펌프, 탱크 구매비
- 고정OPEX: 유지보수비 (CAPEX의 3~5%)
- 변동OPEX: 연료비, 펌핑 에너지비, 냉각비
```

### 주요 발견사항

1. **초기 투자와 운영 비용의 트레이드오프**: Case 1은 높은 초기 투자로 높은 운영 유연성, Case 2는 낮은 초기 투자로 예약 필수

2. **거리의 영향**: Case 2-2 (울산)가 Case 2-1 (여수)보다 16% 빠른 사이클로 **경제성이 우수**

3. **펌프 크기 최적화**: 1,000~1,200 m³/h가 대부분의 경우 최적 (시간 절감과 CAPEX 사이의 균형)

4. **셔틀 크기**:
   - Case 1: 중간 크기 (2,500~5,000 m³)가 최적 (트립 수 vs 자본비 균형)
   - Case 2: 중형 (5,000~25,000 m³)이 최적 (거리와 용량 균형)

5. **20년 장기 관점**: 초기 투자 차이는 할인되지만, 운영 비용 누적이 NPC를 결정하는 주요 요소

## Chapter 1: 프로젝트 개요 및 모델 구조

### 1.1 문제 정의

#### 1.1.1 비즈니스 맥락
Green Corridor는 국제 해운의 탈탄소화를 목표로 하는 이니셔티브입니다. 부산항은 동아시아의 주요 해운 허브이며, 향후 30년 동안:

- **2030년**: 50척의 친환경 암모니아 연료 선박 운영
- **2050년**: 500척으로 증가 (선형 성장 가정)
- **연간 항차 수**: 선박당 12회 (약 30일 간격)
- **항차당 연료량**: 2,158,995 kg (약 3,175 m³, 사용자 요청으로 5,000 m³로 오버라이드)

이들 선박의 연료 수요를 충족시키기 위해 **얼마나 많은 셔틀 선박**과 **어떤 크기의 펌프**가 필요한지를 결정해야 합니다.

#### 1.1.2 의사결정 변수
최적화 모델이 결정해야 할 주요 변수:

1. **x[t]**: t년도에 신규 추가되는 셔틀의 개수 (정수 변수)
2. **N[t]**: t년도의 누적 셔틀 개수 (정수 변수, N[t] = N[t-1] + x[t])
3. **y[t]**: t년도의 연간 벙커링 콜 수 또는 트립 수 (연속 변수)
4. **x_tank[t]**: t년도에 신규 추가되는 저장탱크 개수 (Case 1만, 정수 변수)
5. **N_tank[t]**: t년도의 누적 저장탱크 개수 (Case 1만)

#### 1.1.3 목적함수
최소화 목표: 20년간의 순현재가 (Net Present Cost)

```
Minimize NPC = Σ_t [Discount_Factor(t) × (CAPEX(t) + FIXED_OPEX(t) + VARIABLE_OPEX(t))]
```

여기서:
- **Discount_Factor(t)** = 1 / (1 + 0.07)^(t - 2030) [할인율 7%]
- **CAPEX(t)** = 셔틀 자본비 + 펌프/벙커링 장비비 + 저장탱크 자본비
- **FIXED_OPEX(t)** = 셔틀 고정 운영비 + 펌프 고정 운영비 + 탱크 고정 운영비
- **VARIABLE_OPEX(t)** = 셔틀 연료비 + 펌프 연료비 + 탱크 냉각비

### 1.2 수요 예측 모델

#### 1.2.1 선박 함대 성장

모델은 **2030년부터 2050년까지 20년 동안 선박 함대가 선형적으로 증가**한다고 가정합니다.

```
Vessel_Count(year) = 50 + (500 - 50) × (year - 2030) / (2050 - 2030)
                   = 50 + 22.5 × (year - 2030)
```

**구체적 예시**:
- 2030년: 50척
- 2035년: 162척
- 2040년: 275척
- 2045년: 387척
- 2050년: 500척

#### 1.2.2 연간 수요 계산

각 선박은 **연간 12회의 항차**를 수행하며, 각 항차에서 **2,158,995 kg (약 3,175 m³)의 암모니아 연료**를 적재합니다. 그러나 사용자 요청으로 **벙커링 콜당 5,000 m³**를 기준으로 모델링합니다.

```
Annual_Demand(year) = Vessel_Count(year) × m3_per_voyage × voyages_per_year
                    = Vessel_Count(year) × 3,175 m³ × 12
```

**연도별 수요 예시**:
- 2030년: 50 × 3,175 × 12 = 1,905,000 m³
- 2035년: 162 × 3,175 × 12 = 6,152,400 m³
- 2040년: 275 × 3,175 × 12 = 10,458,000 m³
- 2050년: 500 × 3,175 × 12 = 19,050,000 m³

연간 수요는 **20년 동안 약 10배 증가**합니다.

#### 1.2.3 밀도 변환

암모니아의 저장 조건 밀도를 적용하여 질량을 부피로 변환:

```
m3_per_voyage = kg_per_voyage / (density_storage_ton_m3 × 1000)
              = 2,158,995 / (0.680 × 1000)
              = 3,175.3 m³
```

### 1.3 경제 모델

#### 1.3.1 할인 및 NPV 계산

모든 미래 비용은 **7% 연이율의 할인율**로 현재 가치로 계산됩니다:

```
Discounted_Value(t) = Future_Value(t) / (1.07)^(t - 2030)
```

이는 **시간이 지날수록 비용의 가중치가 감소**함을 의미합니다. 예를 들어:
- 2030년 100만 달러 = 현재 100만 달러
- 2035년 100만 달러 = 현재 71.3만 달러
- 2050년 100만 달러 = 현재 26.3만 달러

#### 1.3.2 연료 가격

- **암모니아 연료 가격**: 600 USD/톤 (기준값)
- **전기 가격**: 0.0769 USD/kWh (펌프 냉각비 계산용)

#### 1.3.3 비용 구성 요소

모델에서 계산되는 모든 비용 항목:

1. **CAPEX (자본비)**
   - 셔틀 선박 구매비
   - 펌프/벙커링 장비비
   - 저장탱크 설치비 (Case 1만)

2. **고정 OPEX (고정 운영비)**
   - 셔틀 선박 유지보수비 (CAPEX의 5%)
   - 펌프 유지보수비 (CAPEX의 5%)
   - 탱크 유지보수비 (CAPEX의 3%)

3. **변동 OPEX (변동 운영비)**
   - 셔틀 연료비 (항해 시 엔진 연료)
   - 펌프 연료비 (펌핑 작업 시)
   - 탱크 냉각비 (암모니아 냉각 유지 비용)

### 1.4 기술 파라미터

#### 1.4.1 엔진 사양

셔틀의 크기에 따라 **최대 연속 정격(Maximum Continuous Rating, MCR)** 파워가 다릅니다:

| 셔틀 크기 (m³) | MCR (kW) | 비고 |
|--------------|---------|------|
| 500 | 1,296 | 기존 데이터 |
| 1,000 | 1,341 | 기존 데이터 |
| 2,000 | 1,429 | 기존 데이터 |
| 3,000 | 1,517 | 기존 데이터 |
| 4,000 | 1,606 | 기존 데이터 |
| 4,500 | 1,650 | 보간값 |
| 5,000 | 1,694 | 보간값 |
| 10,000 | 2,159 | 외삽값 (로그 스케일) |
| 25,000 | 2,981 | 외삽값 |
| 50,000 | 3,867 | 외삽값 |

**MCR 해석**: MCR이 클수록 엔진이 강력하며, 더 많은 연료를 소비합니다.

#### 1.4.2 연료 소비율 (SFOC)

- **SFOC (Specific Fuel Oil Consumption)**: 379 g/kWh
- **의미**: 엔진이 1 kWh의 파워를 생성할 때 379그램의 연료를 소비

셔틀이 한 번의 사이클에서 소비하는 연료:
```
Fuel_per_cycle (ton) = MCR (kW) × SFOC (g/kWh) × travel_time (h) / 1,000,000
```

예: 1,500 kW 엔진, 2시간 항해
```
Fuel = 1,500 × 379 × 2 / 1,000,000 = 1.137 ton
```

#### 1.4.3 펌프 사양

펌프는 벙커링 과정에서 암모니아를 셔틀에서 선박으로 전송합니다:

- **사용 가능한 펌프 용량**: 400, 600, 800, 1,000, 1,200, 1,400, 1,600, 1,800, 2,000 m³/h
- **압력 강하(ΔP)**: 4 bar
- **펌프 효율**: 70%

**펌프 파워 계산**:
```
Pump_Power (kW) = (ΔP [Pa] × Q [m³/s]) / efficiency
                = (4 × 10^5 Pa × Q/3600 [m³/s]) / 0.7
```

예: 1,000 m³/h 펌프
```
Power = (4 × 10^5 × 1000/3600) / 0.7 = 158.7 kW
```

## Chapter 2: Case 1 상세 분석 - 부산항 저장소 기반 시나리오

### 2.1 시나리오 정의

#### 2.1.1 지리적 설정

Case 1은 **부산항에 대규모 저장탱크가 이미 존재**하는 상황을 모델링합니다:

```
[암모니아 공급 터미널]
         ↓
    [부산항 저장탱크]  35,000톤 용량
         ↓
    [소형 셔틀]  500~5000 m³
         ↓
    [부산항 내 선박들]  (항만 내부 이동, ~2시간)
```

**거리 특성**:
- 저장탱크 → 선박 이동: 항만 내부
- **편도 이동 시간**: 약 2시간
- **회전 사이클**: 2시간 편도 + 호스 연결(30분) + 호스 해제(30분) = 약 3시간

#### 2.1.2 운영 로직

Case 1의 핵심 운영 개념:

**개념**: "벙커링 콜(Bunkering Call)" = 한 번에 **5,000 m³를 공급하는 작업**

한 번의 벙커링 콜을 수행하려면:
- 소형 셔틀이 **여러 번 왕복**해야 합니다
- 각 왕복은 셔틀 용량만큼만 전달 가능

**예시 1**: 500 m³ 셔틀로 5,000 m³를 공급하려면
```
필요 트립 수 = ceil(5,000 / 500) = 10 트립

각 트립:
- 저장탱크에서 500 m³ 적재
- 선박으로 이동 (2시간)
- 500 m³ 주유
- 저장탱크로 복귀

10번 반복하여 총 5,000 m³ 공급
```

**예시 2**: 5,000 m³ 셔틀로 5,000 m³를 공급하려면
```
필요 트립 수 = ceil(5,000 / 5,000) = 1 트립

한 번의 항해:
- 저장탱크에서 5,000 m³ 적재
- 선박으로 이동 (2시간)
- 5,000 m³ 주유
- 저장탱크로 복귀

총 1번 왕복하여 5,000 m³ 공급
```

이것이 **Case 1의 근본적 구조**: 크기가 작은 셔틀일수록 **더 많은 트립이 필요**하므로 **더 높은 운영 비용**이 발생합니다.

### 2.2 셔틀과 벙커링 관계

#### 2.2.1 셔틀 크기별 트립 수

| 셔틀 크기 (m³) | 5,000m³ 콜당 트립 수 | 한 해 1척 셔틀의 최대 콜 수 |
|--------------|-----------------|-----------------|
| 500 | 10 | 80 |
| 1,000 | 5 | 160 |
| 1,500 | 4 | 200 |
| 2,000 | 3 | 267 |
| 2,500 | 2 | 400 |
| 3,000 | 2 | 400 |
| 3,500 | 2 | 400 |
| 4,000 | 2 | 400 |
| 4,500 | 2 | 400 |
| 5,000 | 1 | 800 |

**관찰**:
- 셔틀이 클수록 **더 적은 트립**이 필요
- 하지만 **사이클 타임은 더 김** (큰 펌프가 필요하면 펌핑 시간 증가)
- 최적 크기는 비용과 효율성 사이의 **절충점**

#### 2.2.2 연간 벙커링 콜의 의미

**y[t]** = t년도의 연간 벙커링 콜 수

만약 y[2030] = 3,814라는 최적 해를 얻었다면:
```
- 한 해 동안 3,814번의 "5,000 m³ 공급" 작업을 수행
- 각 콜마다 공급되는 선박은 1척 (5,000 m³ = 한 척의 항차 수요)
- 총 공급량 = 3,814 × 5,000 = 19,070,000 m³
- 총 수요 = 50 척 × 3,175 m³ × 12회 = 1,905,000 m³ (맞지 않음)
```

**재해석**: Case 1의 y[t]는 실제로 **"부분적 콜의 누적"**을 의미합니다:
- 실제 해석: 50척 × 12회/년 = 600회의 항차 수요
- 각 항차 = 약 3,175 m³
- 600회 × 3,175 m³ ÷ 5,000 m³ = 381.6 "콜 등가" 단위

따라서 y[2030]은 **"하루 1콜"이 아니라**, **"연간 충족해야 하는 5,000 m³ 단위의 작업량"**으로 해석해야 합니다.

#### 2.3.1 한 번 출항할 때의 주유 프로세스

- 전체 사이클 = 약 8시간

```
연간 최대 사이클 = 8,000 시간/년 ÷ 8시간/사이클 = 1,000 사이클
= 1,000 콜 × 5,000 m³ = 5,000,000 m³/년
```

**제약**: 한 척의 셔틀은 **연간 최대 5,000,000 m³**만 공급 가능
- 2030년 수요: 1,905,000 m³ → 0.38척 셔틀 필요
- 2050년 수요: 19,050,000 m³ → 3.81척 셔틀 필요

### 2.4 저장탱크 역할과 제약

#### 2.4.1 저장탱크의 기능

Case 1의 35,000톤 저장탱크:

```
[공급 터미널] ──(대량 운송)──> [35,000톤 저장탱크] ──(셔틀)──> [선박들]
                              (부산항)                        (연안 운송)
```

**핵심 역할**:
1. **수요 변동 흡수**: 선박들의 시간대별 수요를 평탄화
2. **셔틀 효율화**: 셔틀이 필요할 때 즉시 적재 가능
3. **유연성**: 언제든 선박들이 연료를 요청할 수 있도록 대비

#### 2.4.2 저장탱크 용량 제약

```
Tank_Capacity >= Daily_Peak_Demand × Safety_Factor

35,000톤 × 0.680 ton/m³ = 51,470 m³
```

모델의 제약식:
```
N[t] × Shuttle_Size × 2.0 (안전계수) <= N_tank[t] × Tank_Volume
```

**해석**:
- 모든 셔틀이 동시에 **최대 용량으로 적재**되었을 때
- 그 총량이 탱크 용량의 **2배를 넘지 않아야 함** (안전 여유)

**예**: 셔틀 2척, 각 5,000 m³
```
2 × 5,000 × 2 = 20,000 m³ <= 51,470 m³ (OK)
```

#### 2.4.3 저장탱크의 경제성

탱크 CAPEX 계산:
```
Tank_CAPEX = Tank_Capacity_tons × Cost_per_kg
           = 35,000 × 1,000 × 1.215 USD/kg
           = 42,525,000 USD (약 4.25억 달러)
```

탱크 OPEX:
- 고정비: CAPEX × 3% = 1,275,750 USD/년
- 냉각비: 35,000톤 × 1,000 kg/톤 × 0.0378 kWh/kg × 0.0769 USD/kWh
        = 104,403 USD/년

**총 탱크 비용**: 크기가 작거나 호출 수가 적으면 탱크가 **과잉 투자**가 될 수 있음

### 2.5 비용 구조 분석

#### 2.5.1 셔틀 비용

**CAPEX (자본비)**:
```
Shuttle_CAPEX = 61,500,000 × (Size / 40,000)^0.75

예시:
- 500 m³: 61,500,000 × (500/40,000)^0.75 = 3,894,000 USD
- 5,000 m³: 61,500,000 × (5,000/40,000)^0.75 = 18,917,000 USD
```

**고정 OPEX**: CAPEX × 5%
- 500 m³: 194,700 USD/년
- 5,000 m³: 945,850 USD/년

**변동 OPEX (연료비)**:
```
Shuttle_Fuel_per_cycle = MCR × SFOC × travel_time / 1,000,000
                       = 1,694 × 379 × 2 / 1,000,000
                       = 1.284 ton (5,000 m³ 셔틀, 2시간)

Fuel_cost_per_cycle = 1.284 × 600 = 770.4 USD

연간 1,000 사이클 기준: 770,400 USD/년
```

#### 2.5.2 펌프 비용

**CAPEX**:
```
Pump_Power = (4 bar × 10^5 Pa × Q [m³/h] / 3600) / 0.7
           = 158.73 × Q [m³/h] kW

Pump_CAPEX = Power × 2,000 USD/kW

예시:
- 400 m³/h: 101.2 kW × 2,000 = 202,400 USD
- 1,000 m³/h: 252.8 kW × 2,000 = 505,600 USD
- 1,500 m³/h: 379.2 kW × 2,000 = 758,400 USD
```

**변동 OPEX (펌핑 에너지)**:
```
Pumping_time_per_call = 2 × (5,000 m³ / Pump_capacity)
                      = 2 × (5,000 / 1,000) = 10 시간 (1,000 m³/h 펌프)

Fuel_for_pumping = 252.8 kW × 10 hr × 379 g/kWh / 1,000,000
                 = 0.958 ton per call

Cost_per_call = 0.958 × 600 = 574.8 USD
```

### 2.6 최적화 제약식

#### 2.6.1 수요 충족 제약

```
y[t] × Bunker_Volume_Per_Call >= Annual_Demand[t]
y[t] × 5,000 >= Annual_Demand[t]

예: 2030년
y[2030] × 5,000 >= 1,905,000
y[2030] >= 381
```

이는 **최소 381개의 "5,000 m³ 콜"이 필요**함을 의미합니다.

#### 2.6.2 작업시간 제약

```
y[t] × Trips_per_Call × Cycle_Duration <= N[t] × Max_Annual_Hours

예: 5,000 m³ 셔틀, 1,000 m³/h 펌프
- Trips_per_Call = 1
- Cycle_Duration = 2(이동) + 1(연결) + 5(펌핑) = 8시간
- Max_Annual_Hours = 8,000시간

y[t] × 1 × 8 <= N[t] × 8,000
y[t] <= N[t] × 1,000
```

**해석**: 한 척의 5,000 m³ 셔틀은 **연간 최대 1,000 사이클**을 수행 가능합니다.

#### 2.6.3 일일 피크 제약 (삭제됨, v2.3 이후)

**v2.2까지의 제약식**:
```
Daily_Capacity >= Daily_Demand × Peak_Factor

Daily_Demand = (y[t] / 365) × Bunker_Volume × 1.5
Daily_Capacity = (N[t] × Max_Annual_Hours / Cycle_Duration / 365) × Shuttle_Size
```

**삭제 이유** (v2.3 이후):
1. **작업시간 제약과 중복**: 2.6.2의 연간 총 운항 시간 제약이 이미 일일 변동성을 포함
2. **비현실적 가정**: 1.5배 피크 계수는 근거 부족. 실제 선박 스케줄은 예약 시스템으로 관리
3. **모델 단순화**: 핵심 제약식(수요, 작업시간, 탱크 용량)만 유지로 해석 용이성 증대

**영향**: 모델 결과에는 거의 영향 없음 (작업시간 제약이 더 제한적인 경우가 대부분)

#### 2.6.4 탱크 용량 제약

```
N[t] × Shuttle_Size × 2.0 (안전계수) <= N_tank[t] × Tank_Volume_m3
N[t] × Shuttle_Size × 2 <= N_tank[t] × 51,470 m³
```

**해석**: 모든 셔틀이 동시에 적재되었을 때 탱크 여유는 최소 50% 이상이어야 합니다.

## Chapter 3: Case 2 상세 분석 - 근거지 운송 시나리오

### 3.1 시나리오 정의

Case 2는 **부산항에 저장탱크가 없는** 상황을 모델링합니다. 대신 암모니아를 **원산지(근거지)에서 생산/저장**하고, **필요할 때마다 대형 셔틀로 운송**합니다.

```
[여수 암모니아 생산시설]
```

```
[울산 암모니아 생산시설]

```

#### 3.1.2 근본적 운영 차이

Case 1과 Case 2의 **가장 중요한 차이**:

| 구분 | **Case 1** | **Case 2** |
|-----|----------|----------|

#### 3.1.3 핵심 개념: "한 번의 항해로 여러 선박에게 주유"

Case 2의 운영 개념은 완전히 다릅니다:

**예시: 25,000 m³ 셔틀**
```
한 번의 왕복 여행:
1. 여수 항구에서 25,000 m³ 적재 (15시간 소요)
2. 부산으로 항해 (5.73시간, Case 2-1 기준)
3. 부산에서 계류 중인 선박 5척에게 각각 5,000 m³씩 주유
   - 선박 1: 5,000 m³ (호스 연결 30분 + 주유 5시간 + 호스 해제 30분)
   - 선박 2: 5,000 m³
   - 선박 3: 5,000 m³
   - 선박 4: 5,000 m³
   - 선박 5: 5,000 m³
4. 여수로 복귀 (5.73시간)

결과: 25,000 m³로 5척의 선박을 주유
비용: 1회 셔틀 항해로 5척 서빙
```

이것이 **Case 2의 효율성 메커니즘**: 큰 셔틀 × 한 번의 항해 = 여러 선박 동시 서빙

### 3.2 선박당 주유 패턴

#### 3.2.1 Case 2-2 예시 (울산 → 부산)

**정상 운영 시나리오**:

```
Timeline (50,000 m³ 셔틀 기준):

날짜 1 (월요일):
- 09:00: 울산 항구에 도착하여 50,000 m³ 적재 시작
- 23:00: 적재 완료 (14시간)

날짜 2 (화요일):
- 00:00: 출발
- 02:40: 부산 도착 (25해리 ÷ 15노트 = 1.67시간)
- 03:00: 선박 A와의 호스 연결
- 03:30: 선박 A 주유 시작
- 08:30: 선박 A 주유 완료 (5,000 m³)

- 09:00: 선박 B와의 호스 연결
- 09:30: 선박 B 주유 시작
- 14:30: 선박 B 주유 완료

... (선박 C, D, E도 동일하게 반복)

- 그날 밤 또는 다음날 오전: 울산으로 복귀 시작
- 다음날 오전: 울산 도착

총 사이클 타임: 약 26시간
서빙된 선박: 10척 (50,000 m³ ÷ 5,000 m³)
```

#### 3.2.2 주유 순서와 조정

실제로는 모든 선박이 동시에 부산에 있지 않으므로:

1. **사전 예약 시스템**: 선박들이 24시간 전에 주유 예약
2. **스케줄링**: 셔틀 운영자가 주유 순서 최적화
3. **대기 시간**: 연료를 기다리는 선박 최소화

**모델의 단순화 가정**:
- 모든 선박이 필요할 때 즉시 주유 가능
- 셔틀 크기 = 여러 선박의 수요 합
- 펌프 용량이 충분하면 병렬 주유 가능

### 3.3 셔틀 크기와 선박 서빙 관계

#### 3.3.1 한 번의 항해로 서빙 가능한 선박 수

```
Vessels_per_trip = floor(Shuttle_Size / Bunker_Volume_per_Call)
                 = floor(Shuttle_Size / 5,000)
```

| 셔틀 크기 (m³) | 한 항해당 선박 수 | 연간 서빙 잠재력 (척) |
|--------------|-----------------|-----------------|
| 5,000 | 1 | 200 (연 200회 항해 가정) |
| 10,000 | 2 | 400 |
| 15,000 | 3 | 600 |
| 20,000 | 4 | 800 |
| 25,000 | 5 | 1,000 |
| 30,000 | 6 | 1,200 |
| 40,000 | 8 | 1,600 |
| 50,000 | 10 | 2,000 |

**관찰**:
- 셔틀이 클수록 **한 번에 더 많은 선박 서빙 가능**
- 하지만 **가동 시간이 더 필요** (적재, 펌핑, 복귀)
- 최적점은 **시장의 선박 밀집도**에 달려있음

#### 3.3.2 "트립 수"의 의미 (Case 2에서는 1)

Case 2에서 **가장 중요한 수정 사항**:

```
Case 1: Trips_per_Call = ceil(Bunker_Volume / Shuttle_Size)
        예: 5,000 m³ / 1,000 m³ = 5 트립

Case 2: Trips_per_Call = 1 (항상!)
        대신: Vessels_per_Trip = floor(Shuttle_Size / Bunker_Volume)
```

**이유**:
- Case 1: 작은 셔틀이 **같은 선박에게 여러 번** 왕복
- Case 2: 큰 셔틀이 **여러 선박에게 한 번의 항해로** 공급

따라서 Case 2의 **y[t]는 "항해 수"**이며, 각 항해는:
```
Delivery_Volume = Shuttle_Size (가득 찬 용량)
= Vessels_per_Trip × 5,000 m³
```

### 3.4 운영 타이밍 계산

#### 3.4.1 Case 2-2 울산 예시

**50,000 m³ 셔틀 + 1,000 m³/h 펌프**:

```
1. 적재 (여수/울산):
   Time = 2 × (50,000 m³ / 1,500 m³/h)
        = 2 × 33.3h = 66.7시간
   (왕복 고려: 실제로는 편도 33.3시간, 하지만 계산에서는 2배)

2. 항해 (일편도):
   Time = 1.67시간

3. 주유 (부산):
   Time = 2 × (50,000 m³ / 1,000 m³/h)
        = 2 × 50시간 = 100시간
   (로딩 50시간 + 언로딩 50시간)

4. 귀항:
   Time = 1.67시간

총 사이클: 66.7 + 1.67 + 100 + 1.67 = 170시간

연간 최대 항해: 8,000시간 / 170시간 = 47회
```

**해석**: 한 척의 50,000 m³ 셔틀로 **연간 최대 47회 항해 가능**

#### 3.4.2 사이클 타임의 주요 구성요소

```
Full_Cycle = 2×Travel_Time + 2×Setup_Time + Pumping_Time

Case 2-1 (여수) 기준:
- Travel: 5.73시간 (편도)
- Setup: 0.5시간 (호스 연결/해제)
- Pumping: 2 × (Volume / Pump_Rate)

25,000 m³ 셔틀, 1,000 m³/h 펌프:
= 2×5.73 + 2×0.5 + 2×(25,000/1,000)
= 11.46 + 1 + 50
= 62.46시간

연간 최대 항해: 8,000 / 62.46 = 128회
```


#### 3.5.2 펌프 비용

**문제**: Case 2에서는 펌프 용량이 **병목**이 될 수 있습니다.

```
50,000 m³를 펌프해야 하는데:
- 1,000 m³/h: 50시간 필요 (비현실적, 선박들이 50시간 대기)
- 2,000 m³/h: 25시간 필요
- 4,000 m³/h: 12.5시간 필요 (병렬 펌핑, 2개 펌프)

대형 펌프 CAPEX:
2,000 m³/h 펌프:
Power = 158.73 × 2,000 = 317,460 kW (!)
CAPEX = 317,460 × 2,000 = 634,920,000 USD (!!)

이는 현실적이지 않으므로, 모델은 보통 여러 중간 크기 펌프를 선택:
- 1,000 m³/h × 4개 펌프 = 4,000 m³/h 총 용량
- CAPEX = 505,600 × 4 = 2,022,400 USD
```

**실제 최적해**: Case 2는 **대형 펌프 사용으로 CAPEX 증가**

모델에서는 **Case 2의 탱크가 활성화되지 않음** (부산에만 저장 없다고 가정)

### 3.6 최적화 제약식 (Case 2 특화)

#### 3.6.1 수요 충족 (Case 2)

```
y[t] × Shuttle_Size >= Annual_Demand[t]

예: 2030년, 25,000 m³ 셔틀
y[2030] × 25,000 >= 1,905,000 m³
y[2030] >= 76.2 항해

즉, 한 해에 76회 이상의 왕복 항해 필요
```

**비교**:
- Case 1: y[2030] >= 381 (5,000 m³당 콜 단위)
- Case 2: y[2030] >= 76 (25,000 m³당 항해 단위)

Case 2가 **항해 횟수로는 적지만**, 각 항해의 **거리와 에너지는 훨씬 큼**

#### 3.6.2 작업시간 제약 (Case 2)

```
y[t] × 1 (trips_per_call) × Cycle_Duration <= N[t] × 8,000

예: 25,000 m³ 셔틀, 1,000 m³/h 펌프
y[t] × 1 × 62.46 <= N[t] × 8,000
y[t] <= N[t] × 128.1

즉, 한 척의 셔틀은 연간 최대 128회 항해 가능
```

**2030년 요구사항**: 76회 항해 < 128회 가능
→ **한 척의 셔틀로도 충분**

**2050년**: 790회 항해 필요
→ **6~7척의 셔틀 필요** (790 / 128)

| **여수/울산 근처에서 생산** | Case 2 | 운송 거리 단축 |
| **선박 도착 시간 불규칙** | Case 1 | 탱크의 버퍼링 기능 |
| **수요 예측 가능 (정규 스케줄)** | Case 2 | 사전 계획 효율적 |

## 부록

### A. 모델 가정과 제한사항

#### A.1 주요 가정

1. **선형 선박 증가**: 2030~2050년 선형 증가 (실제는 비선형일 수 있음)
2. **일정한 항차 수**: 모든 연도에 선박당 12회 (계절성 미고려)
3. **완벽한 수급 일치**: 공급이 정확히 수요를 충족 (안전 재고 불고려)
4. **고정 연료 가격**: 600 USD/톤 (실제는 변동)
5. **할인율 일정**: 7% (경제 변화 미반영)
6. **독립적 운영**: 여러 케이스가 경쟁하지 않음 (실제는 할인 가능)

#### A.2 모델 제한사항

1. **기상 조건**: 악천후, 항해 지연 미모델링
2. **장비 고장**: 정기 유지보수 외 예측 불가능 고장 미반영
3. **기술 진보**: 향후 20년 기술 개선 가능성 무시
4. **규제 변화**: 환경 규제, 항만 규칙 변화 미반영
5. **경쟁 효과**: 다른 충전소 출현의 영향 미반영

### B. 모델 파라미터 민감도 분석

#### B.1 할인율 변화의 영향

```
할인율 5%: NPV = NPC × 1.10 (비용 증가)
할인율 7%: NPV = NPC × 1.00 (기준)
할인율 10%: NPV = NPC × 0.92 (비용 감소)

해석: 할인율이 낮을수록 (시간 가치 높음) 초기 투자의 영향 증가
      → Case 1의 탱크 비용이 더 큰 문제
      → Case 2의 상대적 우위 더 커짐
```

#### B.2 연료 가격 변화의 영향

```
연료 $400/톤: 변동 OPEX 33% 감소
연료 $600/톤: 변동 OPEX 기준
연료 $800/톤: 변동 OPEX 33% 증가

영향: 연료비가 전체 비용의 10~15%를 차지하므로
      큰 영향은 아니지만 셔틀 크기 선택에 영향
```

#### B.3 펌프 용량의 중요성

```
펌프 400 m³/h: 사이클 타임 증가 → 셔틀 수 증가 → CAPEX 증가
펌프 1,000 m³/h: 최적점 (사이클-CAPEX 균형)
펌프 2,000 m³/h: 과시설 → CAPEX 증가, 이용률 저하

최적 펌프 크기는 예상 사이클 타임에 따라 결정
```

### C. 모델 수식 요약

#### C.1 목적함수

```
Minimize: Z = Σ_t [Discount(t) × (CAPEX(t) + FixedOPEX(t) + VariableOPEX(t))]

여기서 Discount(t) = 1 / (1.07)^(t - 2030)

CAPEX(t) = Shuttle_CAPEX × x[t] + Pump_CAPEX × 1 + Tank_CAPEX × x_tank[t]
         = (Shuttle_Unit_Cost × x[t]) + Pump_Cost + (Tank_Unit_Cost × x_tank[t])

FixedOPEX(t) = (Shuttle_FixedOPEX × N[t]) + (Pump_FixedOPEX) + (Tank_FixedOPEX × N_tank[t])

VariableOPEX(t) = (Shuttle_Fuel_Per_Cycle × Cycles) + (Pump_Fuel_Per_Call × y[t]) + (Tank_Cooling)
```

#### C.2 주요 제약식

```
1. 재고 균형:
   N[t] = N[t-1] + x[t]  (모든 t > start_year)
   N[start_year] = x[start_year]

2. 수요 충족 (Case 1):
   y[t] × 5,000 >= Annual_Demand[t]

3. 수요 충족 (Case 2):
   y[t] × Shuttle_Size >= Annual_Demand[t]

4. 작업 시간:
   y[t] × Trips_per_Call × Cycle_Duration <= N[t] × 8,000

5. 탱크 용량 (Case 1만):
   N[t] × Shuttle_Size × 2.0 <= N_tank[t] × Tank_Volume

6. 일일 피크:
   Daily_Supply >= Daily_Demand × 1.5
```

```

2. 파라미터 계산

5. 결과 내보내기
```

### E. 용어 정의

| 용어 | 정의 |
|-----|------|
| **Shuttle** | 암모니아를 운반하는 선박 (Case 1: 500~5,000 m³, Case 2: 5,000~50,000 m³) |
| **Call/Trip** | 벙커링 작업 1회 (Case 1: 5,000 m³ 기준, Case 2: 왕복 항해 1회) |
| **Cycle Duration** | 셔틀의 완전한 왕복 사이클 시간 (편도 이동 + 로딩 + 언로딩) |
| **MCR** | Maximum Continuous Rating, 엔진의 최대 연속 정격 파워 (kW) |
| **SFOC** | Specific Fuel Oil Consumption, 연료 소비율 (g/kWh) |
| **CAPEX** | Capital Expenditure, 선박/펌프/탱크 구매 비용 |
| **OPEX** | Operating Expenditure, 운영 및 유지보수 비용 |
| **NPC** | Net Present Cost, 현재가 기준 총 비용 |
| **Annual Demand** | 한 해 동안 필요한 암모니아 총량 (m³) |
| **Discount Factor** | 미래 비용을 현재가로 환산하는 계수 (할인율 7%) |
